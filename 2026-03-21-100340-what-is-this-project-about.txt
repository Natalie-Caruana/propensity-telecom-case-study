     Plan: Building an MLOps Production-Grade System

     Context

     You want to build a production-grade MLOps system using Claude and the
     skills in this repository. The mlops-coding-skills repo defines 7
     sequential skill areas (Initialization вҶ’ Prototyping вҶ’ Industrialization вҶ’
     Validation вҶ’ Automation вҶ’ Collaboration вҶ’ Observability) that together
     cover the full lifecycle from "empty folder" to "monitored, deployed ML
     system". This plan lays out the order of attack, what Claude should do at
     each phase, and how to verify progress.

     ---
     Approach: Follow the Skills in Order

     The 7 skills are explicitly designed as a sequential progression. Each
     builds on the previous. Do not skip phases вҖ” jumping to industrialization
     without initialization leads to structural debt.

     ---
     Phase 1 вҖ” Initialization (mlops-initialization/SKILL.md)

     Goal: Create a clean, reproducible project foundation.

     Steps:
     1. Verify toolchain: uv, git installed
     2. Run uv init <project-name> вҶ’ get pyproject.toml, src/ layout
     3. Add runtime deps: pandas, scikit-learn, pydantic, omegaconf, loguru,
     mlflow
     4. Add dev deps: pytest, pytest-cov, mypy, ruff, bandit
     5. Configure pyproject.toml: ruff rules, mypy strict mode, pytest paths
     6. Create .gitignore, init git, make first commit

     Verify: uv run python -c "import sklearn; print('ok')" succeeds

     ---
     Phase 2 вҖ” Prototyping (mlops-prototyping/SKILL.md)

     Goal: Explore data and build a baseline model in a structured notebook.

     Steps:
     1. Create notebooks/01_prototype.ipynb
     2. Follow mandatory section order: Title вҶ’ Imports вҶ’ Configs вҶ’ Datasets вҶ’
     EDA вҶ’ Modeling вҶ’ Evaluations
     3. Define all hyperparameters and paths as constants at top
     4. Split data before any transformations (avoid leakage)
     5. Build an sklearn.pipeline.Pipeline with ColumnTransformer
     6. Evaluate: metrics, confusion matrix/residuals, feature importance
     7. Tag cells with papermill tags for parameterization

     Verify: Notebook runs end-to-end cleanly with jupyter nbconvert --to 
     notebook --execute

     ---
     Phase 3 вҖ” Industrialization (mlops-industrialization/SKILL.md)

     Goal: Refactor notebook code into a distributable Python package.

     Steps:
     1. Create src/<project_name>/ with three layers:
       - domain/ вҖ” pure functions (transformations, metrics)
       - io/ вҖ” data loaders, model savers (OOP classes)
       - application/ вҖ” pipeline orchestration
     2. Create src/<project_name>/scripts.py with CLI entrypoints
     3. Register CLI in pyproject.toml under [project.scripts]
     4. Add Pydantic config schemas + OmegaConf YAML config files
     5. Add Google-style docstrings and full type hints

     Verify: uv run <project-name> --config configs/default.yaml runs the
     pipeline

     ---
     Phase 4 вҖ” Validation (mlops-validation/SKILL.md)

     Goal: Add quality gates вҖ” types, linting, tests, security.

     Steps:
     1. Run mypy src/ вҶ’ fix all type errors (no untyped Any)
     2. Run ruff check src/ tests/ вҶ’ fix all lint issues
     3. Mirror src/ in tests/ with conftest.py fixtures
     4. Write unit tests for domain functions, integration tests for
     io/application
     5. Run pytest --cov=src --cov-fail-under=80
     6. Add pandera schemas for DataFrame validation at IO boundaries
     7. Run bandit -r src/ вҶ’ fix security issues

     Verify: All checks pass: mypy, ruff, pytest --cov, bandit

     ---
     Phase 5 вҖ” Automation (mlops-automation/SKILL.md)

     Goal: Automate everything вҖ” tasks, hooks, containers, CI/CD, experiment
     tracking.

     Steps:
     1. Create justfile with task modules (tasks/check.just, tasks/package.just,
      etc.)
     2. Set up pre-commit with: ruff, bandit, check-yaml, commitizen
     3. Write Dockerfile using ghcr.io/astral-sh/uv:python3.12-bookworm-slim,
     multi-stage build
     4. Add GitHub Actions: .github/workflows/check.yml (on PR),
     .github/workflows/publish.yml (on release)
     5. Instrument training with MLflow: mlflow.autolog(), log
     params/metrics/artifacts, register model

     Verify: just check passes; Docker builds; MLflow UI shows experiment runs
     at mlflow ui

     ---
     Phase 6 вҖ” Collaboration (mlops-collaboration/SKILL.md)

     Goal: Make the project shareable and team-ready.

     Steps:
     1. Write README.md: badges, hook (why this exists), quickstart
     2. Protect main branch on GitHub (require PR + review)
     3. Add CODE_OF_CONDUCT.md and CONTRIBUTING.md
     4. Set up MkDocs for documentation site
     5. Create CHANGELOG.md following Keep a Changelog format
     6. Add .devcontainer/devcontainer.json for GitHub Codespaces
     7. Establish release workflow: SemVer bump вҶ’ changelog вҶ’ tag вҶ’ GitHub
     Release

     Verify: mkdocs serve runs; pre-commit enforces conventional commits

     ---
     Phase 7 вҖ” Observability (mlops-observability/SKILL.md)

     Goal: Full-stack monitoring, reproducibility, and explainability in
     production.

     Steps:
     1. Fix all random seeds: random, numpy, torch/tensorflow if used
     2. Log git commit hash + uv.lock fingerprint to each MLflow run
     3. Track data lineage with mlflow.log_input() and MLflow Datasets
     4. Add evidently drift detection: data drift + concept drift reports
     5. Log system metrics during training
     6. Add alerting: plyer for local dev, Slack webhook for production
     7. Add SHAP explainability: global feature importance + local explanation
     artifacts logged to MLflow

     Verify: MLflow run page shows: params, metrics, artifacts, data lineage,
     SHAP plots; Evidently drift report generated
